## What the Skill Does

### 功能 1：每日热榜（daily）
- 抓取 https://github.com/trending 页面
- 解析今日最火的仓库（名称、描述、语言、star 数量、今日增长）
- 输出格式：简洁的中文摘要，推送给用户

### 功能 2：月度 Top 10（monthly）
- 抓取过去 30 天的 Trending 数据（按语言分组，循环抓取主要语言）
- 汇总统计每个仓库出现的次数和累计 star
- 选出综合热度最高的 Top 10
- 输出格式：带排名的中文榜单

---

## Technical Requirements

### Input Parameters
- `mode`: `"daily"` | `"monthly"`
- `language`: 语言筛选，如 `"Python"`、`"JavaScript"`、`"Go"`、`"All"`（默认 `"All"`）
- `limit`: 返回数量，默认 10

### Output
- 返回 markdown 格式的文本
- 每条包含：仓库名、描述、语言、今日/总计 star、趋势标签

### Data Source
- 使用 `web_fetch` 或 `requests` 抓取 https://github.com/trending
- 解析 HTML（可用 re 或 BeautifulSoup）
- 不需要 GitHub API token（纯公开页面）

### Scheduling
- `mode=daily` → 每天 UTC 8:00（北京时间 16:00）触发一次
- `mode=monthly` → 每月 1 日 UTC 0:00（北京时间 08:00）触发一次

### 安全要求
- 不发送任何凭证
- 不访问私有仓库数据
- 请求间隔 ≥ 3 秒（避免触发限流）
- 解析失败时要有友好的错误提示

## 注意事项

1. **HTML 解析**：GitHub trending 页面结构可能变化，代码要有容错
2. **限流友好**：两次请求间隔 ≥ 3 秒，mode=monthly 按语言循环时要有 delay
3. **时区处理**：所有时间用 UTC，记录 last_fetched 时间戳
4. **缓存**：月度模式可用 json 文件做简单缓存，避免每次都重新抓 30 天
5. **错误处理**：网络失败时返回上一次的缓存数据（如果有）
